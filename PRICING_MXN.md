## Modelo de Sobrecostos por Tickets

Los sobrecostos se aplican **automáticamente y sin bloqueos**. Si un mes excedes tu límite incluido, continuarás creando tickets normalmente:

| Plan | Límite | Tarifa de Sobrecosto | Ejemplo |
|---|---|---|---|
| **Iniciador** | 250 tickets | $0.70 MXN por ticket | 300 tickets = $1,799 + ($0.70 × 50) = $1,834 MXN |
| **Profesional** | 1,500 tickets | $0.35 MXN por ticket | 2,000 tickets = $5,299 + ($0.35 × 500) = $5,474 MXN |
| **Empresarial** | Ilimitados | $0.00 MXN | Sin cargos adicionales |

**Cómo Funciona:**
1. Tu cuenta rastrea tickets creados cada mes
2. Si superas el límite, se calcula el sobrecosto automáticamente
3. El cargo se incluye en tu siguiente factura
4. Sin sorpresas: recibe advertencias cuando te acercas al límite (80%, 100%)

## Preguntas Frecuentes

**P: ¿Qué pasa si excedo mi límite de tickets incluidos?**
R: No hay bloqueo. Continuarás creando tickets normalmente, pero se te cobrarán sobrecostos automáticamente:
- Iniciador: +$0.70 MXN por ticket adicional
- Profesional: +$0.35 MXN por ticket adicional
- Los sobrecostos se incluyen en tu factura mensual. Si esto ocurre regularmente, te recomendamos actualizar a un plan superior.

**P: ¿Puedo cambiar de plan a mitad del ciclo?**
R: Sí. Los cambios ascendentes entran en vigencia inmediatamente; los cambios descendentes en el próximo ciclo de facturación. Se aplican cargos/créditos prorrateados. Los sobrecostos de tickets se ajustan proporcionalmente.

**P: ¿Qué sucede si excedo mi límite de técnicos?**
R: Te enviaremos un correo de advertencia. Tienes 7 días para actualizar o eliminar usuarios; las cuentas inactivas se eliminan después de 30 días.

**P: ¿Hay contrato?**
R: Iniciador/Profesional: mes a mes. Empresarial: término de 12 meses estándar, negociable.

**P: ¿Ofrecen descuentos anuales?**
R: Sí—10% de descuento al pagar anualmente en Iniciador/Profesional.

**P: ¿Cuál es mi SLA de disponibilidad?**
R: Iniciador/Profesional: 99.5%. Empresarial: 99.9% con respuesta de soporte garantizada <1 hora.

## Notas de Implementación

### Esquema de Base de Datos (Tabla de Precios)

```sql
CREATE TABLE pricing_plans (
  id SERIAL PRIMARY KEY,
  name VARCHAR(50) NOT NULL,
  monthly_price_mxn DECIMAL(10, 2),
  annual_price_mxn DECIMAL(10, 2),
  max_technicians INT,
  max_templates INT,
  max_inventory_items INT,
  features JSONB,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE subscriptions (
  id SERIAL PRIMARY KEY,
  company_id UUID REFERENCES companies(id) ON DELETE CASCADE,
  plan_id INT REFERENCES pricing_plans(id),
  billing_email VARCHAR(255),
  stripe_customer_id VARCHAR(255),
  status ENUM('active', 'paused', 'cancelled', 'past_due'),
  current_period_start DATE,
  current_period_end DATE,
  auto_renew BOOLEAN DEFAULT TRUE,
  currency VARCHAR(3) DEFAULT 'MXN',
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

CREATE TABLE usage_logs (
  id SERIAL PRIMARY KEY,
  subscription_id INT REFERENCES subscriptions(id) ON DELETE CASCADE,
  metric VARCHAR(50), -- 'api_calls', 'storage_gb', 'extra_technicians'
  quantity INT,
  period_date DATE,
  overage_charge_mxn DECIMAL(10, 2),
  created_at TIMESTAMP DEFAULT NOW()
);
```

### Integración de Pagos (Stripe + Conekta)

- **Checkout:** Checkout alojado (Stripe + Conekta para México)
- **Portal de Facturación:** Portal de autoservicio del cliente (gestionar métodos de pago, historial de facturas)
- **Webhooks:** Sincronizar estado de suscripción (`customer.subscription.updated`, `invoice.payment_failed`)
- **Lógica de Reintentos:** Reintentos automáticos de pagos fallidos en día 3, 5, 7
