## Ticket Overage Model

Overages are applied **automatically and without blocking**. If you exceed your included ticket limit in any month, you'll continue creating tickets normally:

| Plan | Limit | Overage Rate | Example |
|---|---|---|---|
| **Starter** | 250 tickets | $0.04 USD per ticket | 300 tickets = $99 + ($0.04 × 50) = $101 USD |
| **Professional** | 1,500 tickets | $0.02 USD per ticket | 2,000 tickets = $299 + ($0.02 × 500) = $309 USD |
| **Enterprise** | Unlimited | $0.00 USD | No additional charges |

**How It Works:**
1. Your account tracks tickets created each month
2. If you exceed the limit, overage is calculated automatically
3. The charge is included on your next invoice
4. No surprises: you'll get warnings as you approach the limit (80%, 100%)

## Frequently Asked Questions

**Q: What happens if I exceed my included ticket limit?**
A: No blockers. You'll continue creating tickets normally, but overages will be charged automatically:
- Starter: +$0.04 USD per ticket above 250/month
- Professional: +$0.02 USD per ticket above 1,500/month
- Charges appear on your next invoice. If this happens regularly, we recommend upgrading to a higher plan.

**Q: Can I change plans mid-cycle?**
A: Yes. Upgrades take effect immediately; downgrades on next billing cycle. Prorated charges/credits apply. Ticket overages are adjusted proportionally.

**Q: What happens if I exceed my technician limit?**
A: We'll email you a warning. You have 7 days to upgrade or remove users; inactive accounts are soft-deleted after 30 days.

**Q: Is there a contract?**
A: Starter/Professional: month-to-month. Enterprise: 12-month term standard, negotiable.

**Q: Do you offer annual discounts?**
A: Yes—10% off when paying annually on Starter/Professional.

**Q: Can I white-label the platform?**
A: Enterprise only. Custom domain, logo, colors, and branded mobile app.

**Q: What's your uptime SLA?**
A: Starter/Professional: 99.5%. Enterprise: 99.9% with guaranteed support response <1 hour.

## Implementation Notes

### Database Schema (Prices Table)

```sql
CREATE TABLE pricing_plans (
  id SERIAL PRIMARY KEY,
  name VARCHAR(50) NOT NULL,
  monthly_price DECIMAL(10, 2),
  annual_price DECIMAL(10, 2),
  max_technicians INT,
  max_templates INT,
  max_inventory_items INT,
  features JSONB,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE subscriptions (
  id SERIAL PRIMARY KEY,
  company_id UUID REFERENCES companies(id) ON DELETE CASCADE,
  plan_id INT REFERENCES pricing_plans(id),
  billing_email VARCHAR(255),
  stripe_customer_id VARCHAR(255),
  status ENUM('active', 'paused', 'cancelled', 'past_due'),
  current_period_start DATE,
  current_period_end DATE,
  auto_renew BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

CREATE TABLE usage_logs (
  id SERIAL PRIMARY KEY,
  subscription_id INT REFERENCES subscriptions(id) ON DELETE CASCADE,
  metric VARCHAR(50), -- 'api_calls', 'storage_gb', 'extra_technicians'
  quantity INT,
  period_date DATE,
  overage_charge DECIMAL(10, 2),
  created_at TIMESTAMP DEFAULT NOW()
);
```

### Payment Integration (Stripe)

- **Checkout**: Hosted Stripe Checkout for new signups
- **Billing Portal**: Self-serve customer portal (manage payment methods, invoice history)
- **Webhooks**: Sync subscription status (`customer.subscription.updated`, `invoice.payment_failed`)
- **Retry Logic**: Auto-retry failed payments on day 3, 5, 7
